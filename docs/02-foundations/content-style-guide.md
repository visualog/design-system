# Foundation Content Style Guide

## 목적
- Foundation 페이지의 `H1 보조 설명`과 `개요 섹션 설명`을 같은 톤과 구조로 유지합니다.
- 사용자(디자이너/개발자)가 페이지를 열자마자 "무엇을, 어떤 순서로 확인하면 되는지" 이해할 수 있게 합니다.

## 문장 템플릿
1. 첫 문장: 이 페이지가 무엇을 정의하는지 설명합니다.
2. 두 번째 문장: 개요에서 무엇을 어떤 순서로 확인할 수 있는지 안내합니다.

### 길이 규칙
- H1 보조 설명은 `정확히 2문장`으로 작성합니다.
- 전체 길이는 `80~130자`를 권장합니다.
- 문장마다 접속어를 최소화해 한 문장 한 의미로 유지합니다.

예시:
- `컬러 시스템은 접근성과 일관성을 기준으로 색상 역할을 정의합니다. 개요에서는 컬러 원칙, 컬러 스케일, 토큰 네이밍 규칙, 사용 가이드를 순서대로 확인할 수 있습니다.`

## 톤 규칙
- 설명형 문체를 사용합니다: `정의합니다`, `확인할 수 있습니다`.
- 과장 표현/마케팅 문구 대신 실무 중심 문장을 사용합니다.
- 한 문장에서 개념은 1개씩 전달합니다.

## H1 언어 정책
- Foundation 페이지의 `H1`은 영문으로 유지합니다.  
  예: `Colors`, `Typography`, `Spacing`, `Layout`
- `H1 보조 설명`과 섹션 설명은 한글을 기본으로 사용합니다.
- 전문 용어는 첫 등장 시 한글 + 영문 병기 후, 같은 페이지에서는 동일 표기를 반복 사용합니다.

## 용어 규칙
- 한글 우선 + 필요 시 영문 병기
- 같은 개념은 모든 페이지에서 같은 단어를 사용

### 표준 용어
- `개요 (Overview)`
- `사용 가이드 (Usage Guide)`
- `시맨틱 토큰 (Semantic Token)`
- `브레이크포인트 (Breakpoint)`
- `간격(gap) / 안쪽 여백(padding)`
- `라인(Line) / 필드(Filled) / 일러스트(Illustration) 아이콘`
- `속성(Property) - 역할(Role) - 변형(Variant) - 상태(State)`
- `인터랙티브 플레이그라운드 (Interactive Playground)`

## 적용 범위
- `/overview`
- `/colors`
- `/typography`
- `/spacing`
- `/layout`
- `/radius`
- `/motion`
- `/icons`
- `/shadows`

## Compact IA 모드 (저콘텐츠 페이지)
- 본문 텍스트가 짧은 페이지는 `compact-ia-template.md` 기준을 우선 적용합니다.
- 원칙:
  - 카드형보다 문서형 블록(`제목 + 본문 + 리스트`)을 기본으로 사용
  - 섹션 간격은 토큰 중심으로 유지하고 개별 `mt/mb` 중복 사용 금지
  - 항목 구분은 `divider` 우선, 장식성 아이콘은 최소화
