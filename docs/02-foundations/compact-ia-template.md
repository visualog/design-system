# Foundation Compact IA Template

## 왜 필요한가
- Expo/Vercel 류 레퍼런스는 본문 텍스트가 긴 문서를 기준으로 설계되어 있습니다.
- 현재 프로젝트는 섹션당 텍스트량이 짧아 동일 여백 규칙을 그대로 쓰면 공백이 커 보입니다.
- 따라서 `정보 구조는 레퍼런스 방식`을 유지하고, `레이아웃 밀도는 컴팩트 모드`로 별도 운영합니다.

## 적용 대상
- Foundation 페이지 중 본문이 짧은 페이지
- 섹션당 설명 2~4문장 + 리스트 3~5개 수준의 문서
- 데모/표/예시 블록이 1~2개인 페이지

## 페이지 기본 구조 (Compact)
1. `H1 (영문)` + `보조 설명 2문장`
2. `핵심 원칙` 섹션
3. `시스템/스케일` 섹션
4. `네이밍/규칙` 섹션
5. `사용 가이드` 섹션 (Do/Don't + 예시)

## 섹션 구성 규칙
- 카드형 컨테이너보다 문서형 블록을 기본으로 사용합니다.
- 기본 패턴: `소제목 + 본문 + bullet` 또는 `소제목 + 표/예시`.
- 항목 구분은 카드 대신 `divider`(경계선) 우선.
- 장식 목적 아이콘은 생략하고 의미 전달 아이콘만 제한적으로 사용합니다.

## 밀도 규칙
- 세로 리듬은 `divide-y + compact row` 패턴으로 통일합니다.
- 불필요한 추가 마진(`mt-*`, `mb-*`)은 기본 간격 토큰과 중복되지 않게 제거합니다.
- 리스트 간격은 기본 `gap-2` 또는 `gap-2.5`를 사용합니다.
- 페이지 마지막 섹션의 추가 하단 마진은 두지 않습니다.

## 카피 규칙 (저콘텐츠 페이지)
- 문장 길이는 짧게, 한 문장 한 메시지 원칙 유지
- 섹션 설명은 1~2문장
- 불필요한 배경 설명 대신 실무 액션 문장 우선

## 구현 체크리스트
- [ ] 섹션 사이 간격이 토큰 외 추가 마진 없이 유지되는가
- [ ] 카드형 박스를 과도하게 사용하지 않았는가
- [ ] 문서형 리스트 블록이 `divider` 기준으로 정렬되는가
- [ ] H1 보조 설명이 2문장 템플릿을 따르는가
- [ ] 같은 유형 페이지에서 제목/설명/리스트 간격이 동일한가
