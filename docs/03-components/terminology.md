# Component Terminology

디자인 시스템 내 `Badge`, `Tag`, `Chip`, `Label` 용어를 아래처럼 고정합니다.

## 1) Label

- 정의: 폼 컨트롤 이름을 설명하는 텍스트
- 예시: `이메일`, `비밀번호`, `검색어`
- 규칙: 폼/입력 문맥에만 사용

## 2) Badge

- 정의: 상태, 등급, 카운트 등 짧은 메타 정보를 표시하는 요소
- 예시: `Stable`, `Beta`, 알림 숫자 `1`
- 규칙: 기본적으로 비인터랙션으로 사용

## 3) Tag

- 정의: 콘텐츠의 분류/주제를 표현하는 토큰
- 예시: `ui`, `form`, `layout`
- 규칙: 분류 목적일 때만 사용

## 4) Chip

- 정의: 선택/삭제/필터 해제가 가능한 상호작용 토큰
- 예시: `Status: Active ×`, `React ×`
- 규칙: 클릭, 선택 해제 같은 액션이 있을 때만 사용

## 5) Meta Badge

- 정의: 파일명, props 수, variants 수 같은 보조 메타 정보 배지
- 예시: `button.tsx`, `4 props`, `5 variants`
- 규칙: 카드/리스트 요약 정보 영역에서만 사용

## 운영 규칙

1. 폼 이름은 `Label`로만 부른다.
2. 상태/카운트/등급은 `Badge`로 부른다.
3. 주제 분류 토큰은 `Tag`로 부른다.
4. 클릭/삭제 가능한 토큰은 `Chip`으로 부른다.
5. 요약 보조 정보는 `Meta Badge`로 부른다.
