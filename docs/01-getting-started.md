# Getting Started

## 대상 독자

- 디자인 시스템 기획자
- UX/UI 디자이너
- 프론트엔드 개발자

## 문서 읽는 순서

1. `02-foundations`: 토큰 정의 확인
2. `03-components`: 컴포넌트 사양 확인
3. `04-patterns`: 화면 패턴 적용 방법 확인
4. `05-governance`: 운영/릴리즈 규칙 확인

## 신규 컴포넌트 추가 기본 흐름

1. 목적/사용 시나리오 정의
2. 아토믹 레벨 분류 (`atom`, `molecule`, `organism`)
3. 접근성/상태/변형 정의
4. 레지스트리 메타데이터 등록
5. 릴리즈 단계 지정 (`experimental`, `beta`, `stable`, `deprecated`)
